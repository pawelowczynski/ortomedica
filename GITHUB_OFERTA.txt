Checklista oferty na GitHubie (jak Trello / Kanban)
===================================================

Repozytorium: pawelowczynski/ortomedica (zgodnie z git remote origin).

1) Zaloguj sie do GitHub CLI (jednorazowo):
   gh auth login

2) Z katalogu glownego projektu (tam gdzie jest oferta-vs-projekt-orthomedica.csv):
   powershell -ExecutionPolicy Bypass -File .\scripts\create-github-oferta-issues.ps1

   Podglad bez tworzenia:
   powershell -ExecutionPolicy Bypass -File .\scripts\create-github-oferta-issues.ps1 -DryRun

   Ponowny import (duplikaty!):
   ... -Force

3) Tablica Kanban:
   https://github.com/pawelowczynski/ortomedica/projects
   Utworz nowy Project (Board), dodaj issues z filtru:
   label:oferta/import-batch

Punkty "Gotowe" i "Wynik biznesowy" sa zamykane automatycznie jako zakonczone; otwarte = do pracy / czesciowo.

Pliki: oferta-vs-projekt-orthomedica.csv, scripts/create-github-oferta-issues.ps1
