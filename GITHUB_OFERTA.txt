Checklista oferty na GitHubie (jak Trello / Kanban)
===================================================

Repozytorium: pawelowczynski/ortomedica (zgodnie z git remote origin).

--- Sposob A (ZALECANY): GitHub Actions — bez logowania gh na komputerze ---

1) Wejdz w repozytorium na GitHubie:
   https://github.com/pawelowczynski/ortomedica/actions/workflows/oferta-import-issues.yml

2) Przycisk "Run workflow" → Run workflow (domyslnie bez ponownego importu).

3) Jesli kiedys uruchomiles import i chcesz drugi raz (UWAGA: duplikaty issues):
   Run workflow → zaznacz "force_import".

4) Tablica Kanban:
   https://github.com/pawelowczynski/ortomedica/projects
   New project → Board → dodaj issues (filtr: label:oferta/import-batch)

--- Zasada: skonczone zadanie = Done na boardzie ---

Na GitHubie kolumna "Done" NIE laczy sie z issue automatycznie, dopoki nie wlaczysz tego w projekcie.

1) Otworz swoj Project (tablica z kartami).
2) Kliknij Workflows / Przeplywy pracy (ikona pioruna lub menu projektu) albo ... przy nazwie projektu → ustawienia automatyzacji.
3) Wlacz przeplyw w stylu: When an issue is closed → Set status to Done (lub: przenies do kolumny Done / Item closed).
   Oficjalny opis: https://docs.github.com/en/issues/planning-and-tracking-with-projects/automating-your-project/using-the-built-in-automations
4) W pracy codziennej: jak zadanie jest skonczone — zamknij Issue (Close issue). Po wlaczonej automatyzacji karta sama wskoczy do Done.

Import CSV juz zamyka issue z etykieta oferta/gotowe i wynik biznesowy — te od razu sa "closed" i powinny ladowac w Done po pkt 1–3.

Pliki: oferta-vs-projekt-orthomedica.csv, scripts/import-oferta-issues.mjs, .github/workflows/oferta-import-issues.yml
